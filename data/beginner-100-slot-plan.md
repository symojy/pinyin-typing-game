# 入門 100 語 — スロット表（案 A）

**字数内訳**：1字 **28** / 2字 **58** / 3字 **12** / 4字以上 **2**（合計 100）

実データは **`scripts/build-beginner-100.mjs`** で生成し、`npm run build:beginner` で `questions-beginner.json` に反映します。  
下表はカテゴリとスロット数の設計用チェックリストです（空欄のままでも可）。

手で編集する場合は各要素で `hanzi[]`・`pinyin[]`・`tones[]` の長さを揃えてください。

---

## 凡例

| 列 | 意味 |
|----|------|
| No. | スロット番号（1〜100） |
| 字 | 1 / 2 / 3 / 4+ |
| カテゴリ | 意味枠（§3 の区分） |
| 漢字 | （記入）ゲーム用の表記 |
| 拼音 | （記入）スペース区切り想定 → JSON では配列に分割 |
| メモ | 出典リストでの順位・HSK など |

## 検算

| 字数 | スロット数 |
|------|------------|
| 1字 | 3+4+7+2+4+3+5 = **28** |
| 2字 | 7+4+2+7+9+5+5+16+3 = **58** |
| 3字 | 1+3+1+2+2+3 = **12** |
| 4字+ | **2** |

## 記入後のチェック（推奨）

- [ ] 中級 `questions-intermediate.json` と **漢字列（`hanzi` 連結）が重複しない**
- [ ] 各語で `hanzi.length === pinyin.length === tones.length`
- [ ] ゲーム内の拼音正規化（ü / v など）と矛盾しない表記

`data/README.md` の入門説明と整合を取る場合は、語数は引き続き **100** で問題ありません。
